Four Hard Problems We Hit — and How We Engineered Around Them

Building an emotionally-aware, IRT-driven tutor on top of an LLM exposed real failure modes. Each one was diagnosed in production-like load and resolved in code before the alpha cut.

03

Cold-start spikes on emotion / sentiment

! Problem

The DistilRoBERTa emotion model took ~3 s to load on first request, creating a long tail on session start and inflating P99 latency every time the Python worker was recycled.

✓ Solution

FastAPI preloads both models at boot via lifespan hooks and runs a single long-lived worker. A dummy warm-up inference runs immediately so the first real request is hot.

chatbot.py FastAPI lifespan port :8001
04

Prompt injection from learner input

! Problem

Curious students tried "ignore previous instructions" attacks, asked for the system prompt verbatim, or tried to coerce the tutor into giving full quiz answers — bypassing the IRT progression entirely.

✓ Solution

Learner messages are sanitized and role-quarantined as untrusted user content. A denylist filter blocks system-prompt extraction patterns; the AI is forbidden by instruction from revealing answers before the learner attempts.
